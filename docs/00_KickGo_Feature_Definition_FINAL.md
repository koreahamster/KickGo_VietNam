# 1. 서비스 개요

KickGo VN은 베트남 아마추어 축구 커뮤니티를 위한 디지털 플랫폼이다.

- 타겟 사용자: 선수 / 심판 / 운동장 관리자
- 종목: 축구(soccer) MVP 핵심 / 풋살(futsal) 향후 확장
- 확장 국가: 베트남 우선, 이후 태국/인도네시아/말레이시아/필리핀
- 기본 언어: vi(베트남어) / ko(한국어) / en(영어)
- 시간대: UTC+7 (베트남)

# 2. 역할 구조

## 서비스 역할 (account_types)

한 사용자는 여러 역할을 동시에 가질 수 있다.

| 역할 | 설명 |
|---|---|
| player | 선수. 팀 활동, 경기 참가, 리그/대회, 소셜 |
| referee | 심판. 경기 배정, 결과 입력, 일당 수령 |
| facility_manager | 운동장 관리자. 운동장 등록, 예약/결제 관리 |

## 팀 내 역할 (team_members.role)

owner와 manager는 동일한 운영 권한을 가진다.

| 역할 | 권한 |
|---|---|
| owner | 팀 소유자. manager와 동일 + 팀원 강퇴 권한 |
| manager | 경기 생성, 결과 수락/거부, 초대 코드 생성, 명단 제출, 회비 관리 |
| captain | 출전 명단 제출 보조 |
| player | 일반 선수 |

강퇴(kick):
- owner만 가능
- 강퇴 시 team_members.status = 'banned'
- 이력 audit_logs 기록

## 앱 관리자 역할 (user_admin_roles)

super_admin이 모든 권한 보유, 하위 역할에 부분 권한 위임 가능.

| 역할 | 권한 범위 |
|---|---|
| super_admin | 전체 권한. 다른 admin 역할 부여/회수 |
| ops_admin | 리그/대회/경기 결과 관리, 분쟁 처리, 운동장 승인, 심판 자격 승인 |
| content_admin | 쇼핑몰 상품 관리, 공지/배너, 신고된 콘텐츠 처리, 제작 주문 관리 |
| support_admin | 사용자 조회/정지/해제, 신고 처리, 결제 환불 지원 |

- 모든 관리자 조치는 audit_logs에 기록
- 관리자 CMS는 웹 기반 별도 구현 (형태/도메인 미정)

# 3. 인증 및 계정

## 3.1 로그인 방식

- Google 로그인
- Apple 로그인
- Facebook 로그인
- 휴대폰 OTP 인증 (필수 — 1인 1계정 정책)
- Email/Password (개발 중 임시 fallback, 프로덕션 제거)

## 3.2 온보딩 흐름

```
소셜 로그인
    ↓
휴대폰 OTP 인증 (본인 확인)
    ↓
공통 프로필 생성 (닉네임, 생년, 지역, 언어, 초기 역할)
    ↓
역할별 상세 온보딩
    ↓
홈 화면
```

## 3.3 공통 프로필 (profiles)

- 닉네임 (display_name)
- 프로필 사진 (avatar_url — Supabase Storage)
- 생년 (birth_year)
- 지역 3단계: 국가 → 도/성 → 구/군 (cascading select, 코드값만)
- 선호 언어 (vi/ko/en)
- 자기소개 (bio)
- 전화번호 (phone, is_phone_verified)
- **프로필 공개 범위**: 사용자 직접 설정
  - public: 누구나 조회 가능
  - members_only: 로그인 사용자만
  - private: 본인만

## 3.4 선수 상세 프로필 (player_profiles)

- 선호 포지션, 주발
- 상의/신발 사이즈
- skill_tier (시스템 관리, 클라이언트 입력 불가)
- reputation_score (시스템 관리, 클라이언트 입력 불가)

## 3.5 심판 프로필 (referee_profiles)

- 평균 평점 (average_rating — referee_ratings 누적)
- 평점 개수 (rating_count)
- (향후) 자격증/레벨

# 4. 선수 기능

## 4.1 홈 대시보드

- 다음 경기 일정 카드
- 최근 알림 요약
- 빠른 접근: 팀 / 경기 / 리그 순위
- 소셜 피드 미리보기

## 4.2 팀

- 팀 생성 (이름, 지역, 설명, 엠블럼 사진)
- 팀 공개 페이지 (팀 소개, 멤버 목록, 최근 성적)
- **팀 공개 범위 설정** (공개/로그인 사용자만/비공개)
- 초대 코드 생성 및 참가
- Zalo 딥링크로 초대 공유
- 팀 내 번호(등번호) 설정
- 팀 탈퇴
- 팀 공지사항 조회 (owner/manager가 작성)

## 4.3 용병 모집

- 팀이 특정 경기에 추가 선수를 모집할 수 있다
- 모집 공고: 경기 날짜, 필요 포지션, 지역, 모집 인원
- 선수가 지원 → 팀 관리자 수락/거부
- 수락된 용병은 해당 경기 출전 명단에 포함 가능
- 용병은 팀 정식 멤버가 아니며, 해당 경기에만 유효
- 용병 경기 기록은 개인 통계에 반영되나 티어 포인트에는 미반영

## 4.4 팀 모집 공고

- 선수가 팀을 찾는 글을 올릴 수 있다
- 팀이 선수를 찾는 공고를 올릴 수 있다
- 지역, 포지션, 티어 필터로 검색

## 4.5 경기 (축구)

- 경기 생성 (홈팀, 원정팀, 일시, 장소, 경기 유형, sport_type)
- **경기 목록: 캘린더형 인터페이스**
  - 경기 카드 표기: 날짜, 상대팀명, 참석 인원수, 경기 유형
  - 예: "3월 22일 | Hue United vs Da Nang FC | 출석 14/22명"
- 경기 상세 조회
- 경기 장소 지도 (Google Maps 연동)
- 경기 일정 외부 공유: React Native Share → Zalo / 카카오 / 링크
- 경기 일정 외부 공유 (Zalo/카카오)
- 출석 투표 응답 (yes/no/late/unknown)
- 출전 명단 제출 (등번호, 포지션)
- 경기 결과 수락/거부 (manager 또는 owner)
- 이의 제기

## 4.6 경기 취소 및 노쇼 정책

- **운동장 예약 연결 경기**: 예약 시 선결제 필수
- 경기 취소 시:
  - 경기 24시간 전 취소: 전액 환불, reputation -5
  - 경기 당일 취소: 50% 환불, reputation -20
  - 노쇼 (취소 없이 불참): 환불 없음, reputation -40
- 노쇼 판정 기준: 경기 시작 후 30분 이내 출석 인원이 최소 인원(7명) 미달
- 노쇼 시 상대 팀에 자동 부전승 처리 (기록 반영)
- reputation_score 기준 하한선 미달 팀은 티어전 자동 매칭 제한

## 4.7 리그 — 티어제 (롤 방식)

- 신규 팀: 브론즈 자동 시작 (참가 신청 없음)
- 티어: 브론즈 → 실버 → 골드 → 플래티넘 → 다이아
- 각 티어 포인트 범위: 0~100점
- 경기 종류별 반영:

| 경기 종류 | 티어 포인트 | 매칭 방식 |
|---|---|---|
| 티어전 | 반영 | 같은 티어 + 같은 지역 자동 매칭 |
| 승급전 | 반영 (승급 결정) | 자동 매칭 |
| 일반 친선 | 미반영 | 수동 (검색/초대) |
| 대회 경기 | 미반영 | 대진표 기반 |

- 티어전 승리: +20~25점 / 패배: -15~20점 / 무승부: ±5점
- 100점 달성 → 승급전 자동 진입
- 승급전: 5판 3승제 (3승 → 승급, 포인트 0 초기화 / 미달 → 유지, 75점 초기화)
- 지역 기준 순위표 (티어별 필터 가능)

## 4.8 대회 (토너먼트)

- 대회 개설 (4팀 기본, 누구나 무료)
- 개최권 구매 시: 8팀/16팀 이상 선택 가능, 대진표 자동 생성
- 팀 참가 등록
- 대진표 조회
- 라운드별 경기 결과 연결
- 우승/준우승 기록

## 4.9 운동장 예약

- 지역/종목/날짜로 운동장 검색
- 가용 구장 및 1시간 단위 시간 슬롯 조회
- 구장 + 시간 선택 후 예약 및 선결제
- 예약 내역 조회/취소
- 운동장 상세 (사진, 시설, 요금, 지도)

## 4.10 MVP 투표 및 경기 기록 카드

### MVP 투표
- 경기 finalized 후 24시간 투표창 활성화
- 투표 포지션: 공격(FW) / 미드필더(MF) / 수비(DF) 각 1명
- 투표 자격: 해당 경기 출전 선수 (1인 1표, 본인 제외) + 심판 추가 1표
- 최다 득표자 → 포지션별 MVP 선정

### FIFA 카드형 경기 기록 카드
- 경기 finalized 후 자동 생성
- 카드 구성:
  - 스코어, 날짜, 팀명, 엠블럼
  - 포지션별 MVP 3인 (공격/미드/수비) + 선수 사진
  - 주요 이벤트 (골/어시/카드 요약)
- 개인 선수 MVP 카드: FIFA Ultimate Team 스타일, 시즌 누적 통계 기반
- SNS 공유: 이미지 저장 → Instagram/Facebook/Zalo/TikTok

## 4.11 소셜 / 커뮤니티

- **쇼츠 업로드**: 축구 기술 짧은 영상 (최대 60초, 최대 1GB)
- **피드**: 팔로우 기반 + 인기 콘텐츠 탐색 탭
- **게시글**: 텍스트 + 이미지/영상
- 좋아요 / 댓글 / 공유
- 선수/팀 팔로우/팔로워
- 해시태그 (#드리블, #프리킥 등)
- 신고 기능 (스팸/욕설/부적절 콘텐츠)
- **차단(Block)** 기능: 특정 사용자/팀 차단

## 4.12 지역 통계

- 지역 선택 후 티어별 팀 순위표
- 지역별 팀 기록 리스트 (승/무/패, 골득실)
- 지역별 선수 기록 리스트 (득점/어시스트/MVP 횟수)
- 티어 필터 (전체/브론즈/실버/골드/플래티넘/다이아)

## 4.13 선수 개인 통계

- 시즌별 누적: 출전 경기, 골, 어시스트, 경고, 퇴장
- MVP 선정 횟수 (포지션별)
- 출석률
- 티어 포인트 변화 이력
- reputation_score 이력

## 4.13-A 선수 이력

- 연도별 소속팀 이력 (어느 팀에서 몇 년에 뛰었는지)
- 시즌별 통계: 출전 경기, 골, 어시스트, 경고, 퇴장, MVP 횟수
- 선수 공개 프로필 페이지에서 이력 조회 가능
- 팀 합류/탈퇴 시 자동 기록 (player_team_history)
- 이적 시스템: 향후 구현

## 4.14 팀 통계

- 현재 티어 및 포인트 (0~100)
- 티어전 성적 (승/무/패, 골득실)
- 팀 내 득점왕, 어시스트왕
- 최근 5경기 결과

## 4.15 쇼핑몰

- 상품 목록 조회 (카테고리: 유니폼/엠블럼 제작, 용품, 개최권)
- 상품 상세 조회
- 장바구니 / 주문 / 결제 (MoMo, ZaloPay)
- 주문 내역 및 배송 상태 조회
- **유니폼 제작 요청**: 팀 로고, 색상, 번호, 이름 커스텀 → 주문 접수 후 수동 처리
- **엠블럼 제작 요청**: 디자인 의뢰 → 주문 접수 후 수동 처리
- **개최권 구매**: 대회 개설 고급 기능 이용권
- **팀 회비 납부**: 팀 내 계좌로 납부 (앱은 기록만 관리)

## 4.16 팀 회비 관리

- 월별 회비 납입 현황 표기 (납입/미납 상태)
- 회비 사용 내역 입력 (owner/manager 입력)
- 팀원이 본인 납입 이력 조회
- 실제 이체는 팀 계좌 직접 이체 (앱은 기록 관리만)

# 5. 심판 기능

## 5.1 프로필 및 설정

- 심판 프로필 등록
- 평균 평점 및 평점 내역 조회
- 경기 이력 조회
- 가용시간 등록/수정/삭제

## 5.2 배정 관리

- 가용 날짜/시간 등록
- 배정 요청 수신 → 수락/거부
- 배정된 경기 목록 조회 (캘린더형)
- 경기 취소 알림 수신

## 5.3 경기 진행

- 경기 시작 전: 양 팀 출전 명단 확인 (등번호/이름)
- 경기 시작 승인 → matches.status = ongoing
- 경기 중 이벤트 기록: 골/어시스트/경고/퇴장 (시간 포함)
- 경기 후: 최종 스코어 + 이벤트 업로드
- MVP 투표 참여 (추가 1표)

## 5.4 결과 확정 및 분쟁

- 결과 업로드 후 양 팀 관리자에게 알림
- 이의 제기 시 24시간 협의
- 24시간 후 미합의 → 심판 결정 자동 반영 (수정 불가)

## 5.5 일당 및 평점

- 일당 지급 기록 조회 (pending/paid/cancelled)
- 양 팀 관리자로부터 받은 평점 조회 (공정성/정확성/태도)
- 평점은 심판 목록에 공개 표시

## 8.1 상품 카테고리

|---|---|
| 유니폼 제작 | 팀 로고/색상/번호/이름 커스텀 제작 의뢰 → 수동 처리 |
| 엠블럼 제작 | 팀 엠블럼 디자인 의뢰 → 수동 처리 |
| 개최권 | 대회 고급 기능 이용권 (가격 미정) |
| 용품 | 축구화, 유니폼, 장갑, 공 등 |

## 8.2 운영 방식

- **앱 내**: 상품 조회, 주문, 결제, 주문 내역 (React Native 네이티브)
- **웹 CMS**: 상품 등록/수정/삭제, 주문 관리, 제작 주문 처리, 배송 정보 입력 (content_admin)

## 8.3 유니폼/엠블럼 제작 주문 흐름

```
선수/팀이 앱에서 제작 요청 접수
    ↓

    ↓
결제
    ↓
content_admin이 웹 CMS에서 주문 확인
    ↓
수동 처리 (제작 업체 연락)
    ↓
제작 완료 → 배송 정보 입력
    ↓
앱 주문 상태 업데이트 (제작중→배송중→완료)
    ↓
고객 알림 발송
```

- 결제 확정: 서버 Webhook 기반 (클라이언트 불가)

# 10. 경기 결과 확정 프로토콜 — 축구 전체 흐름

## 10.1 시작 전

```
경기 생성 (팀 관리자)
    ↓
심판 배정 요청 → 심판 수락
    ↓
출전 명단 제출 (팀 관리자/captain)
    ↓
심판이 양 팀 명단 확인 (등번호/이름)
    ↓
경기 시작 승인 → ongoing
```

## 10.2 경기 중

- 심판이 이벤트 실시간 기록: 골/어시스트/경고/퇴장 (시간 포함)

## 10.3 경기 후

```
심판 최종 스코어 + 이벤트 업로드
    ↓
양 팀 관리자(manager/owner)에게 알림
    ↓
수락 또는 이의 제기 (24시간 이내)
    ↓
양 팀 수락 → finalized
한 팀 거부 → disputed (24시간 협의)
24시간 무응답 → auto_finalized
    ↓
분쟁 24시간 후 미합의 → 심판 결정 자동 반영
```

## 10.4 확정 후

- league_standings 자동 재계산 (리그 경기인 경우)
- skill_tier 포인트 반영 (티어전인 경우)
- reputation_score 반영
- 선수 개인 통계 반영
- MVP 투표창 활성화 (24시간)
- 경기 기록 카드 자동 생성
- 심판 일당 기록 생성

## 10.5 심판 평점

- 경기 finalized/auto_finalized 후 양 팀 관리자가 심판 평점 등록
- 항목: 공정성/정확성/태도/종합 (각 1~5점)
- 팀당 1회, referee_profiles.average_rating에 누적

## 티어 구조

| 티어 | 설명 |
|---|---|
| 브론즈 | 신규 팀 자동 시작 |
| 실버 | |
| 골드 | |

| 다이아 | 최상위 |

## 포인트 규칙

- 각 티어 범위: 0~100점
- 티어전 승리: +20~25점 (상대 티어 가중치)
- 티어전 패배: -15~20점
- 무승부: ±5점
- 친선경기: 미반영

## 승급전

- 100점 달성 → 승급전 자동 진입
- 같은 지역 동일 티어 팀과 5판 3승제 자동 매칭
- 3승: 승급 (포인트 0 초기화)
- 3승 미달: 현재 티어 유지 (포인트 75 초기화)

## 매칭 우선순위

1. 같은 티어 + 같은 지역 (최우선)
2. 같은 티어 + 인접 지역 (대기 30분 초과 시)
3. 인접 티어 + 같은 지역 (대기 60분 초과 시)
