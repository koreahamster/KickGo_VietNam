# KickGo VN — 화면 명세서 (Screen Specification)

문서 버전: 1.3
작성일: 2026-03-20
목적: 각 화면의 구성 요소, 진입 경로, 나가는 경로, 주요 상태를 정의한다.
변경 이력: v1.3 — 역할 전환 사이드 드로어로 변경. 운동장 관리자 대시보드를 구장×시간 2D 그리드 방식으로 재설계. 팀 채팅 텍스트+이미지 MVP 확정. v1.2 — 심판/운동장 관리자 탭 추가. 팀 채팅 탭 추가 (텍스트+이미지 MVP). 역할 전환 UI(상단 드롭다운) 추가. 알림 딥링크 구조 정의.
      Codex가 이 문서를 참고하여 화면 컴포넌트를 정확하게 구현한다.

읽는 법:
- [진입] = 이 화면으로 오는 경로
- [나가기] = 이 화면에서 이동하는 경로
- [컴포넌트] = 화면에 표시되는 UI 요소
- [상태] = 로딩/에러/빈 상태 처리 필요 여부
- [권한] = 해당 역할만 접근 가능 (없으면 공통)

# 1. 인증 플로우

## 1-1. 스플래시 화면 (Splash)
경로: `/(splash)`

[진입] 앱 최초 실행
[나가기]
- 로그인 세션 있음 → 홈 화면 (탭 내비게이션)
- 세션 없음 → 로그인 화면

[컴포넌트]
- KickGo 로고
- 로딩 인디케이터

---

## 1-2. 로그인 화면 (Login)
경로: `/(auth)/login`

[진입] 스플래시, 로그아웃 후
[나가기]
- 소셜 로그인 성공 + 프로필 있음 → 홈 탭
- 소셜 로그인 성공 + 프로필 없음 → 온보딩 (전화번호 인증)
- 회원가입 버튼 → 동일 흐름 (소셜만 지원)

[컴포넌트]
- KickGo 로고 + 슬로건
- Google 로그인 버튼
- Apple 로그인 버튼
- Facebook 로그인 버튼
- Zalo 로그인 버튼
- 전화번호 로그인 링크 (OTP)
- 개인정보처리방침 동의 문구 (하단 고정)

[상태] 로딩: 소셜 로그인 처리 중 스피너

---

## 1-3. 전화번호 인증 화면 (Phone Verify)
경로: `/(auth)/phone-verify`

[진입] 소셜 로그인 성공 후 (신규 사용자), 전화번호 로그인 선택
[나가기]
- 인증 성공 + 프로필 없음 → 프로필 생성
- 인증 성공 + 프로필 있음 → 홈 탭

[컴포넌트]
- 국가 코드 선택 (+84 베트남 기본)
- 전화번호 입력 필드
- OTP 발송 버튼
- OTP 6자리 입력 필드 (자동 포커스)
- 재발송 타이머 (60초)
- 재발송 버튼

[상태]
- 전송 중 로딩
- 잘못된 OTP 에러 메시지
- 만료 에러 메시지

---

## 1-4. 개인정보 동의 화면 (Consent)
경로: `/(auth)/consent`

[진입] 전화번호 인증 성공 직후 (신규 사용자만)
[나가기] 동의 완료 → 프로필 생성

[컴포넌트]
- 개인정보처리방침 동의 체크박스 (필수, 스킵 불가)

- 개인정보처리방침 전문 링크
- 동의하고 계속하기 버튼

경로: `/(onboarding)/create-profile`

[진입] 동의 완료 후
[나가기] 저장 완료 → 역할 선택

- 자기소개 입력 (최대 200자, 선택)

[상태]

- 에러 필드 하이라이트

- 각 카드: 아이콘 + 역할명 + 한 줄 설명
- 복수 선택 가능 (체크마크 표시)

[진입] 역할 선택에서 선수 선택

[컴포넌트]

- 상의 사이즈 선택 (XS~3XL)

- 완료 버튼

## 2-4. 온보딩 튜토리얼 (Tutorial)
경로: `/(onboarding)/tutorial`

[진입] 역할별 상세 온보딩 완료 후 (최초 1회)
[나가기]
- 다음 버튼 → 다음 슬라이드
- 완료 / 스킵 → 홈 탭

[컴포넌트]
- 슬라이드 페이저 (5단계)
- 각 슬라이드: 일러스트 + 제목 + 설명
- 하단 도트 인디케이터
- 스킵 버튼 (우상단)
- 다음 / 완료 버튼

슬라이드 내용:
1. 팀 만들기
2. 경기 잡기
3. 심판 배정
4. 리그 티어
5. 기록 카드 공유

# 3. 선수 — 탭 구조 정의

## 하단 탭 5개 (확정)

| 탭 | 아이콘 | 내용 |
|---|---|---|
| 홈 | house | 대시보드 (다음 경기, 알림 요약, 지역 순위, 피드 미리보기) |
| **팀** | users | 내 팀 목록 → 팀 상세 (경기/멤버/공지/회비/통계 내부 탭) |
| 소셜 | play | 쇼츠 피드, 게시글 피드 |
| 탐색 | compass | 검색, 용병/팀 모집, 운동장 예약, 쇼핑몰, 지역 통계 |
| 내 프로필 | person | 내 프로필, 선수 이력, 설정 |

> 경기는 독립 탭이 아닌 팀 탭 내부에 위치. 경기는 반드시 팀 컨텍스트 안에 존재.

# 3-A. 선수 — 홈 탭

## 3-1. 홈 화면 (Home)
경로: `/(tabs)/home`

[진입] 탭 내비게이션 홈 탭 탭
[나가기]
- 다음 경기 카드 탭 → 팀 탭 > 팀 상세 > 경기 탭 (해당 경기 상세)
- 팀 공지 탭 → 팀 탭 > 팀 상세 > 공지 탭
- 지역 순위 탭 → 탐색 탭 > 지역 통계
- 알림 아이콘 탭 → 알림 목록

[컴포넌트]
- 상단바: 로고 + 알림 아이콘 (배지)
- 다음 경기 카드 (없으면 "경기를 만들어보세요" 빈 상태)
  - 상대팀명 / 날짜·시간 / 장소 / 출석 N/M명
- 내 팀 공지 최신 1건
- 지역 티어 순위 요약 (내 팀 현재 순위)
- 최근 경기 결과 카드 (최대 3건)
- 소셜 피드 미리보기 (최신 2개)

[상태]
- 팀 없음: "팀에 참가하거나 만들어보세요" CTA
- 스켈레톤 로딩

---

# 4. 선수 — 팀 탭

## 4-0. 팀 목록 (Team List)
경로: `/(tabs)/team`

[진입] 탭 내비게이션 팀 탭 탭
[나가기]
- 팀 카드 탭 → 팀 상세 페이지
- + 버튼 → 팀 생성
- 팀 없음 상태 → 팀 만들기 / 초대코드로 참가 CTA

[컴포넌트]
- 내가 속한 팀 카드 목록
  - 팀 엠블럼 + 팀명 + 지역 + 현재 티어 뱃지 + 내 역할
- + 팀 만들기 버튼
- 초대코드로 참가 버튼

[상태]
- 팀 없음: "아직 팀이 없습니다" + CTA 2개 (만들기 / 참가)

---

## 4-1. 팀 상세 (Team Detail)
경로: `/(team)/[teamId]`

[진입] 팀 목록 카드 탭, 홈 카드 탭, 검색 결과, 초대 딥링크
[나가기]
- 내부 탭으로 이동 (홈 / 경기 / 멤버 / 공지 / 회비 / 통계)
- 설정 아이콘 → 팀 설정 (manager/owner만)

[컴포넌트]
- 팀 엠블럼 + 팀명 + 지역
- 현재 티어 뱃지 + 포인트 바 (0~100)
- 내부 탭 네비게이션 (7탭: 홈 / 경기 / 멤버 / 공지 / 채팅 / 회비 / 통계)
- 팀원이 아닌 경우: 참가 신청 버튼만 표시

---

## 4-2. 팀 홈 탭 (Team Home)
경로: `/(team)/[teamId]` (default 탭)

[컴포넌트]
- 다음 경기 카드 (없으면 "경기를 만들어보세요")
- 현재 티어 + 포인트 바
- 초대 코드 공유 버튼
- 용병 모집 공고 바로가기

---

## 4-3. 경기 캘린더 (Match Calendar)
경로: `/(team)/[teamId]/matches`

[진입] 팀 상세 > 경기 탭
[나가기]
- 경기 카드 탭 → 경기 상세
- + 버튼 → 경기 생성 (manager/owner만 표시)

[컴포넌트]
- 상단: 월 이동 화살표 + 현재 월 표시
- 캘린더 그리드 (월간 뷰)
  - 경기 있는 날: 도트 인디케이터 표시
- 선택된 날짜 아래 경기 목록
- 경기 카드 1건:
  - 날짜·시간
  - 홈팀 vs 원정팀 (팀 엠블럼 포함)
  - 참석 인원 N/M명
  - 경기 유형 뱃지 (티어전 / 친선 / 대회)
  - 경기 상태 뱃지 (예정 / 진행 중 / 완료)

[상태]
- 빈 날짜 탭: "이 날은 경기가 없습니다"
- 로딩: 스켈레톤

---

## 4-2. 경기 상세 (Match Detail)
경로: `/(team)/[teamId]/matches/[matchId]`

[진입] 팀 상세 > 경기 탭 > 캘린더 카드 탭, 홈 다음 경기 카드 탭
[나가기]
- 출석 투표 → 인라인 처리 (모달)
- 장소 보기 → 지도 화면
- 외부 공유 → 시스템 공유 시트
- 명단 보기 → 명단 화면
- 결과 수락 → 결과 확인 모달

[컴포넌트]
- 상단: 팀 vs 팀 (엠블럼 + 스코어)
- 경기 정보 섹션: 날짜/시간, 장소, 경기 유형
- 장소 보기 버튼 (Google Maps)
- 일정 공유 버튼 (Zalo / 카카오 / 링크)
- 출석 현황: 응답별 인원수 (출석 N / 결석 M / 미응답 K)
- 내 출석 투표 버튼 (yes / no / late)
- 출전 명단 섹션 (명단 확정 후)
- 이벤트 기록 섹션 (경기 중/후)
- 결과 섹션 (finalized 후): 스코어 + 이벤트 요약

[상태]
- 경기 전: 출석 투표 활성
- 경기 중: 실시간 이벤트 (Supabase Realtime)
- awaiting_confirmation: 수락/거부 버튼 표시 (manager/owner만)
- finalized: 결과 카드 + MVP 투표 버튼

---

## 4-3. 경기 생성 (Create Match)
경로: `/(match)/create`

[권한] team manager / owner만 접근
[진입] 캘린더 + 버튼, 팀 상세 경기 생성 버튼
[나가기] 생성 완료 → 생성된 경기 상세

[컴포넌트]
- 내 팀 선택 (홈팀 자동)
- 상대팀 검색 + 선택
- 날짜·시간 피커
- 장소 입력 (Google Maps 검색)
- 경기 유형 선택 (티어전 / 친선 / 대회)
- sport_type 선택 (축구 고정, MVP에서)
- 심판 필요 여부 토글
- 생성 버튼

---

## 4-4. 출전 명단 (Match Roster)
경로: `/(match)/[matchId]/roster`

[진입] 경기 상세 > 명단 탭
[나가기] 저장 → 경기 상세

[컴포넌트]
- 팀별 탭 (홈팀 / 원정팀)
- 멤버 목록 (체크박스로 출전 선택)
- 등번호 입력 필드 (선수별)
- 포지션 선택 (선수별)
- 용병 추가 버튼
- 저장 버튼 (manager/owner/captain만)
- 심판 확인 완료 배지 (심판 전용)

---

## 4-5. MVP 투표 (MVP Vote)
경로: `/(match)/[matchId]/mvp`

[진입] 경기 상세 > MVP 투표 버튼 (finalized 후 24h)
[나가기] 투표 완료 → 경기 상세

[컴포넌트]
- 상단: 투표 마감까지 남은 시간 타이머
- 공격 MVP 선택 섹션
  - 출전 FW 선수 목록 (사진 + 이름 + 등번호)
  - 선택 시 하이라이트
- 미드필더 MVP 선택 섹션
- 수비 MVP 선택 섹션
- 본인 제외 자동 처리
- 투표하기 버튼

[상태]
- 이미 투표 완료: 결과 미리보기 표시
- 투표 마감: 최종 결과 표시

---

## 4-6. 경기 기록 카드 (Match Card)
경로: `/(match)/[matchId]/card`

[진입] 경기 상세 > 기록 카드 보기, 알림에서 진입
[나가기]
- SNS 공유 → 시스템 공유 시트
- 이미지 저장 → 갤러리 저장

[컴포넌트]

  - 스코어 + 날짜 + 팀명 + 엠블럼
  - 포지션별 MVP 3인 (사진 + 이름 + 스탯)
  - 주요 이벤트 요약 (골/어시/카드)
- 저장 버튼
- 공유 버튼 (Instagram / Facebook / Zalo / TikTok)

# 5. 선수 — 팀 탭 나머지 화면

## 5-1. 팀 멤버 탭 (Team Members)
경로: `/(team)/[teamId]/members`

[진입] 팀 상세 > 멤버 탭
[나가기]
- 멤버 카드 탭 → 해당 선수 공개 프로필

# 13. 운동장 관리자 화면 (Facility Manager Mode)

## 13-1. 운동장 관리자 홈 — 통합 대시보드
경로: `/(facility-tabs)/home`

[핵심 설계 원칙]
운동장이 여러 개여도 "지금 어디가 비고 어디가 찼는지"를
전환 없이 한 화면에서 파악할 수 있어야 한다.
→ 구장×시간 2D 그리드 뷰 (호텔 객실 현황판 방식)

[컴포넌트]
- 상단 요약 카드 4개 (오늘 예약 건수 / 빈 슬롯 수 / 대기 중 / 오늘 수익)
- 운동장 전환 탭 (상단, 운동장 복수 등록 시 활성)
  - 탭 전환 시 해당 운동장 그리드로 교체 (드롭다운이 아닌 탭)
- 구장×시간 2D 그리드
  - 행(row): 구장 (A구장 / B구장 / C구장...)
  - 열(col): 시간대 (06:00 ~ 22:00, 1시간 단위)
  - 셀 색상:
    - 초록: 빈 슬롯 (예약 가능)
    - 파랑: 예약 확정
    - 노랑: 대기 중 (수락 필요) — 강조 표시
    - 회색: 차단됨 (운영 불가)
  - 셀 탭 → 해당 예약 상세 또는 슬롯 액션 바텀시트
- 대기 중 예약 요청 목록 (그리드 아래)
  - 카드: 구장명 + 시간 + 예약자 + 금액 + 수락/거부 버튼
  - 수락/거부 즉시 그리드 셀 색상 업데이트

[슬롯 탭 액션 (바텀시트)]
- 빈 슬롯: 슬롯 차단하기 / 닫기
- 예약 확정: 예약자 정보 / 취소 처리 / 닫기
- 대기 중: 예약자 정보 / 수락 / 거부 / 닫기
- 차단: 차단 해제 / 닫기

[기술 구현]
- 가로 스크롤 (시간대 많을 경우)
- Supabase Realtime 구독 (예약 상태 실시간 반영)

# 13. 화면 간 공통 규칙

## 내비게이션 구조
- 하단 탭 5개: 홈 / 경기 / 소셜 / 탐색 / 프로필
- 탭 내에서 스택 내비게이션 (Expo Router)
- 모달: 결제 / 신고 / 확인 다이얼로그 / MVP 투표

## 빈 상태 (Empty State) 공통 규칙
- 모든 목록 화면에 빈 상태 UI 필수
- 빈 상태 구성: 아이콘 + 안내 문구 + CTA 버튼 (있는 경우)

## 로딩 상태 공통 규칙
- 리스트: 스켈레톤 카드 3개 표시
- 단일 데이터: 스피너
- 버튼 액션: 버튼 내 인라인 스피너 + 비활성화

## 에러 상태 공통 규칙
- 네트워크 에러: 재시도 버튼 포함 에러 메시지
- 권한 에러: "접근 권한이 없습니다" + 뒤로가기
- 폼 에러: 필드 단위 인라인 에러 메시지

## 권한별 UI 표시 원칙
- manager/owner 전용 버튼은 해당 역할에게만 렌더링
- 권한 없는 사용자의 직접 URL 접근은 서버(RLS)에서 차단

## 딥링크
- 경기 공유: `kickgo://match/{matchId}`
- 팀 초대: `kickgo://team/{teamId}/join/{inviteCode}`
- 대회 참가: `kickgo://tournament/{tournamentId}`
